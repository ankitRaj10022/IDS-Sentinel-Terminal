# IDS Product Terminal

This repository contains an Intrusion Detection System product terminal built around the bundled IDS CSV datasets. The main Windows product entrypoint is `terminal.cmd`. It works without Conda, Docker, pandas, scikit-learn, or TensorFlow, so a user can inspect traffic data and generate reports with the Python already available on this machine.

The older FastAPI/Docker automation app and ML training code are still present. The product terminal is the recommended first interface for Windows users.

## What This Product Does

- Reads the bundled `kddtrain.csv` and `kddtest.csv` traffic datasets.
- Builds a self-learning IDS profile from labeled CSV rows.
- Analyzes traffic rows and assigns risk, prediction, confidence, and behavior family.
- Exports downloadable CSV and JSON reports.
- Learns again from reports generated by the terminal itself.
- Provides Unix-like shell commands for file inspection and searching.
- Provides defensive network triage commands such as local ports, netstat, DNS, process list, hash, file scan, IOC storage, and IOC hunting.
- Stores command outputs and indexes in a cache folder for review.

## Safety And Scope

This is a defensive IDS and investigation terminal. Use network probing commands only on systems and networks you own or are authorized to test.

This product does not install firewall rules, block packets, exploit systems, bypass authentication, hide processes, or run malware. The `malware` command reports malware-like behavior indicators inferred from IDS features. The bundled CSV labels are binary `normal` and `attack`, not confirmed malware-family labels.

## Quick Start

Open PowerShell in the repository root:

```powershell
cd "C:\Users\danny\Desktop\Intrusion-Detection-Systems-master\IDS-MachineLearning"
```

Run the product status command:

```powershell
.\terminal.cmd status
```

Build or refresh the self-learning model:

```powershell
.\terminal.cmd learn
```

Analyze the first 5,000 rows of the test CSV and export results:

```powershell
.\terminal.cmd scan kddtest.csv --limit 5000
```

List generated reports:

```powershell
.\terminal.cmd reports
```

Open the interactive product terminal:

```powershell
.\terminal.cmd
```

Inside the shell, type:

```text
help
status
traffic
attacks
ports 20
cache 20
exit
```

```text

```

Each `scan` or `export` creates:

```text

traffic_analysis_<timestamp>.json
```

The self-learning model is stored here:

```text
automation/product/self_learning_model.json
```

Command output cache is stored here:

```text
automation/product/cache/commands/
```

Dataset/index metadata cache is stored here:

```text

```

The terminal automatically prunes command cache artifacts after 500 files.

## External Datasets

The built-in catalog points at large public IDS datasets. These are not downloaded automatically because some are multi-GB datasets.

Catalog command:

```powershell
.\terminal.cmd datasets
```

Current catalog sources:

```text
CIC-IDS2017
https://www.unb.ca/cic/datasets/ids-2017.html

UNSW-NB15
https://research.unsw.edu.au/projects/unsw-nb15-dataset

UNSW-NB15 Hugging Face mirror
https://huggingface.co/datasets/lacg030175/UNSW-NB15
```

Import an already downloaded CSV:

```powershell
.\terminal.cmd import C:\path\to\dataset.csv --name external.csv
.\terminal.cmd index automation/product/imports/external.csv --all
```

Download a public CSV URL into the import folder:

```powershell
.\terminal.cmd download https://example.com/dataset.csv --name external.csv
```

The downloader has a 2 GB safety limit per file. For very large datasets, download with a browser or dataset manager, then use `import`.

## Interpreting Results

`risk_score` is the model-estimated probability that a row matches the learned attack profile.

Risk levels:

```text
low       risk_score < 0.55
medium    risk_score >= 0.55
high      risk_score >= 0.75
critical  risk_score >= 0.90
```

Behavior families are heuristic labels inferred from the 41 IDS features:

```text
normal
dos_flood
probe_scan
credential_abuse
privilege_escalation
malware_like_activity
payload_or_exfiltration
network_attack
```

The model type is a pure-Python streaming Gaussian profile. It is intentionally lightweight so the product can run in this Windows environment without data-science packages.

## Troubleshooting

If PowerShell blocks scripts, use the CMD launcher:

```powershell
.\terminal.cmd status
```

If Python is not found, install Python 3 and reopen PowerShell.

If `terminal.exe` is missing, use `terminal.cmd`. On this machine, Windows Defender blocked and removed the locally compiled unsigned `terminal.exe`. The source remains at:

```text
tools/TerminalLauncher.cs
```

If the old Conda activation script fails, install Miniconda or Anaconda first. The script now stops clearly if Conda is missing:

```powershell
.\activate_ids_env.ps1
```

## Development Notes

- Main product code is in `ids_app/product_terminal.py`.
- Manual launchers are `terminal.cmd` and `terminal.ps1`.
- The product terminal avoids heavy dependencies to keep Windows startup reliable.
- The old ML modules still require pandas, numpy, scikit-learn, TensorFlow, and related packages.
- Keep generated exports and cache under `automation/product/`.
- Do not commit Python `__pycache__` files.
